bjf debug模块
该模块主要提供在屏幕上输出debug信息的函数或功能
具体使用效果可参考示例页面
方法和对象列表：
  • bjf.openDebug
  • bjf.debug
  • bjf.cdebug
方法和对象说明：
bjf.openDebug
开启debug开关，否则使用bjf.debug将不能得到任何输出
                
bjf.openDebug();  //开启debug开关
bjf.debug(x)
在屏幕上输出需要打印的变量
变量可以是任意类型，但是如果是多维数组或多维对象，将只递归到第二层
如果变量是Erron对象，打印信息将会用红色字体标出，如果支持stack的话，还将提示错误发生在js文件的哪一行
如果上次的打印信息已经超出屏幕高度了，则下次打印时会自动清空之前打印的类容
参数可变，可输出任意个参数，将依次打印
                
bjf.openDebug();
bjf.debug(a);       //在屏幕上打印变量a，依赖于openDebug。
bjf.debug(a, b, c); //在屏幕上打印变量a,b,c，依赖于openDebug。

try {
    //xxx
}
catch (e) {
    bjf.debug(e);   //如此用法会追踪错误信息，并用红色字体标出。
}
bjf.cdebug(x)
清除屏幕之前的打印关系后，在屏幕上输出需要打印的变量
参数可变，可输出任意个参数，将依次打印
                
bjf.openDebug();
bjf.cdebug(a, b, c); //清除之前的打印信息之后，在屏幕上打印变量a,b,c，依赖于openDebug。
