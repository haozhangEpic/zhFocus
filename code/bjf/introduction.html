欢迎使用BJF框架

简介

BJF框架是一款TV前端框架，框架封装了TVweb前端开发中最常用的部分，例如光标处理，播放器封装，以及遥控器按键识别等。

框架解决了各型号机顶盒之间的兼容性差异，支持所有型号的高清机顶盒，并根据机顶盒对HTML5的支持度，自动选用高性能方法与接口，以及在低端盒子上的自动降级。

框架按模块封装成不同的子对象，各个对象之间不依赖或轻度依赖，页面可根据自身情况选择不同模块。框架整体结构清晰，代码量少，压缩后仅仅58k大小。占用资源少，方便集成。

框架整体采用utf-8编码，但是在项目整体无中文内容输出，理论上对页面编码无要求。

框架提供的所有子模块和方法，命名空间都以"bjf"开头，对页面命名空间不做干预。

下载

请去项目管理平台上根据目标git分支构建相应的工程，生成bjf.js文件

版本说明

BJF框架每次发布，会产生3种不同格式的版本，分别是：基础版、基础压缩版和全量版

基础版：根据依赖关系，把各个模块(除了plyerUI)代码集合并生成的完整框架代码，是一个带版本号的js文件。

基础压缩版：将基础版的代码进行压缩，得到压缩版。压缩版代码不可读，但是体积小性能高。

全量版：包含plyerUI模块的版本，因为plyerUI自带一套播控皮肤，所以该版本下载的是一个zip包，包内包含image文件夹以及全量版的js文件和压缩过的全量版js，一般页面不需要播控，所以无需下载全量版。

引入

在页面中引入BJF框架，需把框架js文件放在body的最底部引入，具体示例如下：

                
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
</head>
<body>
    ...
    <!-- 因框架需要对页面性能和环境做检测，因此需要将框架js引入放在body的底部 -->
    <script src="bjf.1.0.3.min.js"><script>
</body>
</html>

注意事项

页面z-index层级约定，页面应该使用小于10000的z-index层级。大于10000的由框架的debug，光标等模块使用

使用BJF框架后，BJF将会监听页面的window.onload方法，因此页面不能私自覆盖onload的监听。如果需要监听页面加载完成，可采用bjf.onload(call_back_func)的方式。详见Remoter模块介绍

使用BJF框架后，BJF将会监听document.onkeydown，因此页面不能私自覆盖onkeydown的监听。如果需要监听按键事件，可使用Remoter模块的方法。详见Remoter模块介绍
