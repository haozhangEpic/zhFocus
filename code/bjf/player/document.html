bjf player模块
该模块主要提视频播控功能，UI自适应。
可支持多个视频的播控，可以实现一个页面上有多个视频窗口，或者大小视频窗口的业务，需要注意机顶盒是否能支撑多个视频流播放；
UI自适应：进程条，字体等会按窗口比例缩放；
切换焦点窗口，可对焦点窗口进行播放控制；
该模块(将)兼容所有高清IPTV机顶盒，OTT暂未测。
一个完整的播控模块一般包括：player.js、playerUI.js、自定义页面。

该模块的主要功能有：
倍速快进退：按“快进”、“快退”键，视频会按2/4/8/16/32倍速快进退；
拖放快进退：按一次“左右”方向键，每按键一次视频快进退30秒；
暂停：按“暂停”键视频暂停，再次按“暂停”键或者“返回”键，视频继续播放；
暂停时，按左右键可以拖放快进退；
一键到头：按“上页”键，视频到头从头播放；
一键到尾：按“下页”键，视频到尾结束，调用页面自定义的结束画面。
返回：正播情况下按“返回”键，视频暂停，调用页面自定义的返回画面；再次按下“返回”键，继续播放，隐藏页面自定义的返回画面。
结束：提供接口反馈。

该模块在IPTV机顶盒和OTT机顶盒上的差异：
OTT：
无静音控制；
无音量控制；
无声道控制；
不能禁止本地UI；
以上控制按键发送后，该模块不会有响应。
快进退只支持拖放模式；
IPTV：
快进退可选择支持拖放和倍速模式；

demo.html

与v1.0版本的主要区别：
屏幕大小变化UI自适应： 进程条，字体等会按窗口比例缩放
多屏UI独立 视频窗口拥有独立的一组UI
多屏播控独立： 可支持多屏播放，切换焦点窗口，可进行焦点窗口播控操作
键值监听： 页面自定义键值监听，发送给bjf player需要响应的键值
提供功能插件： 针对音量渐强、日志，提供插件案例

通用接口：
bjf.player.getFocusVideo
获取焦点视频窗口的名字；

……

bjf.player.video(name,id,type)
用于建立视频对象；
传参：
name是新建窗口的命名，如果同名视频已经建立了，则不会响应；
id是STB本地播放器实例的instanceID，如果这个id的播放器实例已经存在，则新建的视频对象会绑定这个id的STB本地媒体播放实例，适用于多个页面切换，不关闭窗口，只是切流的情况；
针对华为V6平台，如果是建立PIP流的视频，则type传参"PIP"，如果是马赛克流，则type传参"mosaic"，普通的流则不需要传参type；

MediaPlayer对象是对于终端播放直播、点播、即时时移、回看、音乐等这些媒体内容的能力进行的封装，并将其播放的基本行为抽象为MediaPlayer的相关方法以实现播放，快进，跳转，暂停，停止等操作。
STB应该能够支持同时创建至少2个MediaPlayer对象。 
                
var vod = new bjf.player.video("vod");
bjf.player.clearVirtualEvent
在开始新视频前调用，可以清除虚拟事件。仅适用于华为Q22机顶盒，别的机顶盒不要用。
在V6平台测试，发现在视频开始前，有以前视频的虚拟事件残留，导致混乱，才添加了该接口，但生产环境未发现该现象：

bjf.player.onKey
用于页面的键值响应。
响应键值：左，右，快进，快退，确认，上页，下页，暂停，静音，音量+，音量-，声道，返回

*.setDisplayArea
设置视频显示位置和大小的属性。
属性为 宽, 高, 上边距, 左边距，设置后并不会立即改变视频的显示，需要调用*.videoShow()才能改变视频的显示位置和大小，如果稍后会调用*.start()，则不需要调用*.videoShow()，*.start()会自动调用*.videoShow()。
框架会比较page-view-size的尺寸，当宽度大于等于page-view-size的宽，或者高度大于等于page-view-size的高，则全屏播放；
当宽度小于50，或者高度小于50时，宽度或者高度等于50。 
                
var vod = new bjf.player.video("vod");
……
vod.start();
……
vod.setDisplayArea(600,400,0,0);//设置宽600px，高400px，上边距为0px，左边距为0px的视频
vod.videoShow();
*.setPageViewSize
设置页面可见区域的位置和大小属性，目前主要控制音量图标的显示位置，如果视频窗为小窗时，设置*.setPageViewSize(1280,720,0,0)，可以以全屏的版式范围显示音量图标，而不是局限在视频窗口内。
如不设置，默认为视频窗口大小。
属性为 宽, 高, 上边距, 左边距，需要设置在*.start()之前。
bjf会比较视频窗口的尺寸，页面可见区域的位置和大小不会小于或遮挡视频窗口； 
                
var vod = new bjf.player.video("vod");
……

*.setPlayTime

直播和点播设置时间都为秒，假设n秒,点播是从00:00:00开始的n秒开始播放，直播，假设具有时移功能的频道，为当前的时移开始时间的n秒开始播放。
需要在*.start()前设置。
相似功能也可以通过player.custom.playmode()与*.playByTime()的组合来实现。 
                
var time = 100;
var beginFlag = 0;
                
var vod = new bjf.player.video("vod");

……
vod.start();

*.setHttpTime(t1,t2,t3)
设置视频在HTTP拖放模式下的两个拖放步长。
t1是单次的步长，t2是连续快进退的步长，t3是两次快进退按键的间隔时间，如连续按键间隔小于t3，则认为连续按键。
t1必须小于t2，默认t1=15,t2=30,t3=1000,t1,t2单位为秒,t3为毫秒，t3为1000ms，是经过多次测试的合理值，建议不要小于600ms。 
                
var vod = new bjf.player.video("vod");
vod.setHttpTime(15,30);
vod.start();
*.setInitVol(str)
设置视频窗口初始的声音状态。
传参是："on"|"off"。
"on"：初始时有声状态；"off"：初始时静音状态；不设置，即默认，则框架不处理声音，由机顶盒的视频窗口原始状态决定。
需要在*.start()前设置。 
                
var vod = new bjf.player.video("vod");
vod.setInitVol("on"); //初始有声状态
vod.start();
*.setMosaicVoice(str)
设置马赛克视频窗口的声音状态。
传参是："on"|"off"。
"on"：有声状态；"off"：无声状态；不设置，即默认无声。
需要在*.start()前设置。 
                
var vod = new bjf.player.video("vod");
vod.setMosaicVoice("on"); //有声状态
vod.start();
*.start(time,str1,str2,ottext)
设置视频开始播放。
可选参数：time，vod时长/频道时移时间
由于有极少数盒子通过mp接口取不到频道的时移时间、或者vod的时长，所以提供在start接口提供参数time，让后端取得时移时间、vod时长来赋值给框架，单位为秒;之后框架则调用这个传入的时移时间使用，框架不再请求时移时间。不需要设置时移时间时，传null；
str1是字符串，如果是"noInit",则框架开始视频前，不会调用初始mp，这种情况比较适用于，同一个视频窗切换视频时，不希望它改变已有的视频设置时使用。
str2是字符串，如果是"seek",则框架使用playByTime起流，如果不是，则使用fromStart从头起流后，playByTime跳转，使用前请确保机顶盒支持playByTime起流。
ottext：OTT播放所需的额外参数，比如优酷等项目需要。格式为object，其下所有的key终将做为setPlayParams的参数送给OTT播放器
ottext：示例：{Vid:'xxx', Fid:'xxx'}

*.videoHide

bjf.player.visLog.js
在页面中引入该文件，即可在视频开始、结束时以触发img的src方式发送visLog日志。
当视频开始的时候，会发送playstart的日志。
当视频播放到尾的时候，会发送playend的日志。

里面有4个函数，页面端需要进行调用：
visLog.setUrl(url);
用于设置vis日志服务器url
设置Vis日志服务器的url，会同时在visLog和localStorage里面都存储一次，但由于localStorage不支持跨域，所以存在取不到的风险，如果在visLog和localStorage都没有取到这个值的话，就不会发送日志。
所以，每次在需要发送日志的页面都Vis日志服务器的url是最为保险的做法，而只在开机页面存储一次，存在跨域取不到的情况。

visLog.setParaFixed(obj);
用于设置单页面固定的上下文的环境参数，参数传递的是对象

visLog.setParaUnfixed(obj);
用于设置在连播，或者节目单环境下，类似mediacode这类在单页面会变化的参数，参数传递的是对象

visLog.send(str);
参数为："playend"|"playstart"
用于设置在连播，或者节目单环境下，跳转节目时，或者在跳转离开播控页面，或者release视频时，即无法自然触发bjf的视频开始结束日志时，页面主动发送当前视频日志。

注意：如需要发送日志，visLog.setUrl(url),visLog.setParaFixed(obj),visLog.setParaUnfixed(obj)需要在*.start()前设置好，否则不会发送日志。
注意：如引用了多个功能点，注意bjf.player.plugin.playmode不要相互覆盖，可以添加到一个bjf.player.plugin.playmode里面。
附加功能是为了满足业务的个性化需求而开发的，可以根据实际需求进行修改。
